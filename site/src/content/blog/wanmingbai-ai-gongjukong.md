工具控的意思是：即使没有任何收益，折腾工具本身就能让你快乐。

Twitter 上的 AI 大博主、一人公司的开发者，几乎都有这个特征。

因为现在的 AI 是不成熟的，产出是不确定的，工具链和配套设施都是缺的。

没有工具控的那股劲，根本撑不过最枯燥的那段配置环节。

<div class="my-10"></div>

## 时间成本有多大？简单算一笔账

以我为例，光是把基础设施搭起来，就得搞定：

**一、一个稳定的网络环境。** 纯净 IP 节点，自建 VPS 或国际 eSIM，选一个。

**二、稳定可靠的境外支付。** 香港卡基本不行。Wise 和虚拟卡，一定会被风控。Apple 礼品卡安全性骤降，得需要欧美合规银行卡挨个尝试。每张卡还得绑 Apple Pay、Google Pay 一张张试，看哪张不被拒。

**三、Claude、Gemini、GPT 的会员。** 要么去薅大厂犄角旮旯的羊毛（谷歌云赠金、Gemini 学生优惠），要么就得在一堆中转站里甄别哪家 token 没掺水。

搞定这些，恭喜你，可以开始用 Agent 了。

然后你会发现，真正的坑现在才开始。

<div class="my-10"></div>

## 跑 Agent 才真正让人崩溃

**1. 风控和封号。**

Claude Code 跑在本地，VPS 一旦失效会怎样？

本地和云端同时跑、IP 不一致，Anthropic 会不会封号？

这些问题没人替你回答，只能自己一次次试出来。

**2. Token 消耗失控。**

龙虾出来之后，token 消耗量直接起飞。

我高峰期一晚上跑了 1.5 亿 token，直接被 Coding Plan 限额。

任务跑到一半断掉、输出戛然而止，那种心态，真的会爆炸。

**3. 上下文窗口爆炸。**

长任务跑到一半，Agent 突然"失忆"：忘了你最初的需求，开始按自己的理解胡乱改代码。

什么时候 /compact、什么时候 /clear、什么时候干脆重开会话，CLAUDE.md 怎么写才能让它在长任务里不跑偏——这些没有官方答案，全靠自己踩。

**4. Agent 幻觉。**

让它修一个小 bug，它能把整个模块重写。

让它跑测试，它会告诉你"全部通过"，你一看，测试文件根本没生成。

要不要给 --dangerous 模式，给了之后怎么兜底？Git 是最后的安全网，当然前提是它没顺手把 .git 也改了。

**5. 国产模型怎么搭。**

Claude 太贵，必须配国产模型用。

Opus 4.7、GLM、千问、Kimi，分别擅长什么？Sub-agent 怎么分工？

这套搭配现在没有标准答案，全是自己跑出来的经验。

<div class="my-10"></div>

以前我很不理解那些博主在折腾什么。没有任何提效，作用不明显，投入时间和金钱成本巨高，为什么一个个乐此不疲？

我现在知道了，他们本身就是工具控。在 AI 出来之前，这批人往往折腾的是 NAS、笔记软件和云服务器，只不过现在换成了 AI。

不过，真沉浸进去，倒也能感受到快乐。

我最兴奋的一天就是从早上 6 点起来，挨个测试模型性能，解决网络问题，逐个调试。在晚上 11 点的时候，终于跑通了多 Agent 协作，搭建了一个完整的工作流，能将我日常 5 个小时的工作压缩到 10 分钟解决。

那天烧掉了我小 1000 块钱，但真的很快乐，比任何消费旅游都要快乐。

<div class="my-10"></div>

想跟上 AI，其实真的很简单。不需要搞关系，不需要复杂的职场技巧。

**足够纯粹，本身就是最好的助力。**
